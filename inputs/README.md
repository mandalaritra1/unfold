# Inputs

All input pickles are **gitignored** (`*.pkl`) — this README is the only tracked
record of the expected layout. The directory tree is organized by
`channel / observable / [tag]`, matching the spec/runner conventions in
`src/unfold/tools/`.

```
inputs/
  zjet/
    rho/
      original/     # pre-JEC-fix set  (default rho tag — RHO_SPECS["original"])
      fixed_jec/    # JEC-fixed set     (RHO_SPECS["fixed_jec"])
    mass/           # placeholder — regenerate via the mass input-maker (empty)
    validation/     # detector-level Data/MC validation set (Run 2)
  dijet/
    rho/            # 2018 channel rho inputs
  trijet/
    rho/            # 2018 channel rho inputs
  _archive/         # inactive / superseded; not read by any live workflow
    gluonjetmass/   # rho+mass set from an inactive git branch
    dijetHists_QCD_herwig_all.pkl
```

## zjet — rho (`inputs/zjet/rho/<tag>/`)

Consumed by `RHO_SPECS[<tag>]` in `unfold.tools.unfolder_core`, driven from
`notebooks/unfolder_v4_rho.ipynb`. Two tags exist; `original` is the current
default (`RHO_SPEC`). The `fixed_jec` set additionally carries the
`minimal_rho_data_<era>.pkl` per-era data files.

| File | Sample | Notes |
|------|--------|-------|
| `pythia_all.pkl` | PYTHIA8 (merged) | nominal MC, all eras merged |
| `pythia_<era>.pkl` | PYTHIA8 per-era | 2016, 2016APV, 2017, 2018 |
| `jk_pythia_<era>.pkl` | PYTHIA8 jackknife | per-era response-matrix MC stats |
| `jk_data_all.pkl` | data jackknife | merged |
| `data_all.pkl` | data (merged) | reconstructed only |
| `herwig_all.pkl` | HERWIG (merged) | model/closure |
| `pythia_reweighted_all.pkl` | PYTHIA8 reweighted | herwig-model fallback |
| `st_all.pkl` | single top | background |
| `minimal_rho_data_<era>.pkl` | data per-era | `fixed_jec` only |

## zjet — mass (`inputs/zjet/mass/`)

**Currently empty (placeholder).** The mass pickles must be regenerated by the
mass input-maker workflow before the zjet mass path (`MASS_SPEC`) can run.
Expected filenames mirror the rho set with a `mass_` prefix
(`mass_pythia_all.pkl`, `mass_data_all.pkl`, `mass_herwig_all.pkl`,
`mass_jk_data_all.pkl`, `mass_pythia_<era>.pkl`, `mass_jk_pythia_<era>.pkl`).

## zjet — validation (`inputs/zjet/validation/`)

Run 2 detector-level Data/MC validation, read by
`notebooks/data_mc_rho_fancy.py` and `scripts/compute_dy_purity.py`:
`validation_pythia_<era>.pkl`, `validation_data.pkl`,
`validation_backgrounds_all.pkl`, `validation_st_all.pkl`.

## dijet / trijet — rho (`inputs/<channel>/rho/`)

2018 channel rho inputs, discovered by
`unfold.tools.rho_channel_inputs.discover_rho_channel_files` and run via
`scripts/run_rho_unfolding.py`:

- `minimal_rho_<channel>_data_<year>.pkl` — reconstructed histograms only.
- `minimal_rho_<channel>_mg_pythia8_<year>.pkl` — reco + gen + response.
- (`minimal_rho_dijet_herwig_2018.pkl` present for dijet but not required.)

The discovery helper also falls back to the legacy flat `inputs/<channel>/`
layout when no `rho/` subdir is present.

## _archive

`gluonjetmass/` is a rho+mass set from an inactive branch and is **not** part of
the current rho+mass × zjet/dijet/trijet matrix. `dijetHists_QCD_herwig_all.pkl`
was a stray top-level file. Nothing in the live code reads `_archive/`.
